# LIFF画面設計

**作成日：** 2026-04-25  
**ステータス：** 草案

---

## 基本方針

- LIFF = Next.js（Vercel）のページをLINEアプリ内ブラウザで開いたもの
- `@line/liff` SDKでLINE認証・ユーザー情報取得
- v1はグループメンバー専用の画面のみ（閲覧専用ページはv2）

---

## 画面一覧

| 画面 | URL | アクセス条件 |
|---|---|---|
| 初回セットアップ | `/setup` | LINE認証済み・グループ未所属 |
| グループ参加 | `/join?token=xxx` | LINE認証済み |
| タイムライン（メイン） | `/timeline` | LINE認証済み・グループメンバー |
| アラート設定 | `/settings` | LINE認証済み・グループメンバー |

---

## 画面遷移

```
Bot友達追加
  └── Bot が /setup リンクを送信
        └── /setup（グループ作成 + 犬登録）
              └── 完了 → /timeline

招待URLをタップ
  └── /join?token=xxx（グループ参加確認）
        └── 参加完了 → /timeline

リッチメニュー「タイムライン」
  └── /timeline（メイン）
        └── 設定ボタン → /settings
```

---

## 各画面の詳細

### /setup（初回セットアップ）

**用途：** Botを友達追加した最初のユーザーがグループ・犬を登録する

**フォーム項目**
- グループ名（テキスト入力）
- 犬の名前（テキスト入力）
- 犬の誕生日（日付ピッカー）

**送信後**
- API: `POST /api/v1/groups` → グループ作成
- API: `POST /api/v1/dogs` → 犬登録 + alert_settings自動生成（pee/poop 各4時間）
- `/timeline` にリダイレクト

---

### /join（グループ参加）

**用途：** 招待URLを受け取った家族がグループに参加する

**表示内容**
- グループ名（「〇〇家に参加しますか？」）
- 参加ボタン

**送信後**
- API: `POST /api/v1/groups/join` → トークン検証・自動承認でメンバー追加
- `/timeline` にリダイレクト

---

### /timeline（メイン）

**用途：** 状態確認 + ケア記録のタイムライン表示

#### 状態サマリー（上部・スクロールなしで見える）

```
おしっこ  2時間前
うんち    5時間前
散歩      今日2回
ごはん    12:00
```

- `recorded_at` をもとにフロントが計算して表示
- うんちが設定時間を超えたら色で警告（アラート設定値と比較）

#### タイムライン（スクロール）

各ケアをカード形式で時系列表示

```
┌──────────────────────────┐
│ 🦴 散歩（ロングコース）  │
│ お母さん · 14:00         │
└──────────────────────────┘
┌──────────────────────────┐
│ 🍚 ごはん               │
│ お父さん · 12:00         │
└──────────────────────────┘
┌──────────────────────────┐
│ 💧 おしっこ             │
│ お母さん · 10:30（2時間前）│
└──────────────────────────┘
```

- 表示する `recorded_at` は「○時」と「○時間前」を併記
- APIから `GET /api/v1/dogs/:dog_id/care_records` で取得

#### ナビゲーション

- 右上に設定アイコン → `/settings`

---

### /settings（アラート設定）

**用途：** 排泄アラートの間隔を変更する

**表示・編集項目**

| 項目 | デフォルト |
|---|---|
| おしっこアラート | 4時間 |
| うんちアラート | 4時間 |

- スライダーまたはセレクトボックスで時間を選択
- 変更後: `PATCH /api/v1/dogs/:dog_id/alert_settings`
