# 要件定義書

**アプリ名：** manmaru-shiba  
**作成日：** 2026-04-25  
**ステータス：** 草案

---

## 1. 解く課題

家族全員で犬の世話をするとき、「最後におしっこ出したのいつ？」「今日散歩いった？」「夕飯あげた？」という情報が家族LINEに散らかる。情報が一元管理されておらず、誰が何をしたか把握しにくい。

---

## 2. ユーザー

| ユーザー種別 | 権限 | 説明 |
|---|---|---|
| 同居家族 | 書き込み・読み込み | メインユーザー |
| 離れた家族 | 読み込みメイン | 「今日のうちの子」を見たい |
| 犬友 | 読み込み専用（v2） | 写真シェアのみ |

---

## 3. 機能要件

### v1（MVP）

#### 3-1. LINEから記録する
- 食事・排泄・散歩をLINEから記録できる
- アプリを開かずにLINEだけで操作できることが必須
- 実装方式：LINE Bot（1対1トーク）＋ Flex Message ＋ LIFF

**LINE Bot操作フロー**
- 家族全員がBotを友達追加して1対1でトーク
- リッチメニュー（トーク画面下部の固定ボタン）で文字入力ゼロで記録できる
- ボタン例：[散歩した] [ごはん] [状態確認]
- 「状態確認」タップ → BotがDBから最新情報を取得してFlex Message（カード）で返信
- Flex Messageのカード上からも記録ボタンを操作できる
- 「タイムライン」ボタンでLIFF（全記録画面）を開く

#### 3-2. タイムライン表示
- 家族全員の記録をタイムライン形式で表示する
- 誰がいつ何をしたかが一目でわかる

#### 3-3. 排泄アラート
- 最後の排泄記録から設定時間が経過したら通知する
- デフォルト：4時間
- 時間はユーザーが設定画面から変更できる（個体差・年齢差に対応）

## 4. 非機能要件

| 項目 | 内容 |
|---|---|
| 運用コスト | 0円（無料枠のみで構成） |
| 対象 | 柴犬に絞る（他犬種は対象外） |
| 認証 | LINE認証（LIFFのユーザー情報を利用） |

### 4-1. 認証・アクセス制御

| アクセス方法 | 認証 | 権限 |
|---|---|---|
| LINE内（LIFF） | LINE認証（自動） | 書き込み・読み込み |
| 外部ブラウザ | なし（公開URL） | 読み込みのみ |

**家族グループ管理**
- 最初にBotを友達追加したユーザーが家族グループを作成
- 招待URLを既存の家族LINEに貼って共有
- 家族が招待URLからBotを友達追加 → 同じグループに参加

**v2以降**
- 外部ブラウザでもLINEログインで認証 → 書き込み可能に

### 4-2. セキュリティ実装上の注意

- LINE Channel Secret / Channel Access Token は必ず環境変数で管理
- `.env` をgitにコミットしない（`.gitignore` に追加必須）
- Render・Vercelの環境変数設定画面で値を登録する

## 5. 競合と差別化

| サービス | 弱点 |
|---|---|
| ペットノート+ | LINE連携なし |
| うちっ子ログ | LINE連携なし |
| DogLog（海外） | 日本語・LINE未対応 |

**差別化ポイント：** アプリを開かなくてもLINEから記録できる

---

## 6. 技術スタック

| 役割 | 技術 | 選定理由 |
|---|---|---|
| バックエンド | Rails API only | 既存知識・GoodJobでゼロ円構成 |
| フロントエンド | Next.js / TypeScript | 既存知識・Vercel無料枠 |
| LINE連携 | LIFF + Webhook | ミニアプリより審査不要・個人開発に適切 |
| ジョブ処理 | GoodJob | PostgreSQLのみで完結・Redis不要 |
| DB | PostgreSQL（Supabase） | 無料枠 |
| バックホスティング | Render.com | Rails対応・無料枠 |
| フロントホスティング | Vercel | Next.js対応・無料枠 |
| スリープ対策 | UptimeRobot | Renderの無料プランスリープ防止 |
