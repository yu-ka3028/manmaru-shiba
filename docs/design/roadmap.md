## Phase 2：MVP計測 🔍

**MVPが動いたら必ず計測してからv1.1に進む。**

### 計測項目

| 項目 | 確認方法 | 判断基準 |
|---|---|---|
| webhookの応答時間 | Railsログ（`Completed 200 OK in XXXms`） | 1000ms超えたら要注意・2000ms超えたらGoodJob検討 |
| LINEのリトライ発生有無 | LINE Developersのwebhookログ | リトライが起きていたらタイムアウトしている |
| Renderのスリープ発生有無 | UptimeRobotのダウン通知 | 通知が来ていたら対策強化 |

### 判断
- 問題なし → Phase 3へ
- 応答時間が危険域 → GoodJob導入をIssueに起票してPhase 3へ
- スリープ頻発 → UptimeRobotの間隔調整

### 記録場所
`docs/scrap/YYYY-MM-DD_mvp-measurement.md` に計測結果を残す。
